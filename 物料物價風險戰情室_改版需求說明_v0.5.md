# 第二章　工程重估頁

## 2.1　最重要的前提：本頁定位變更

**現況**：模擬版左欄有「選擇工程」，內含三個寫死的工程樣本（鐵路高架化橋梁工程 / 跨溪鐵路鋼構橋梁新建工程 / 鐵路橋梁耐震補強工程）。每個樣本帶入四項資料：工程名稱、工程類型、原始編列時間、各材料用量。右欄的三張 KPI、瀑布圖、成本走勢圖、建議預算，全部由「選中的工程」推導而來。

**問題**：實際 Power BI 環境中**沒有工程主檔可供導入**，使用者不可能在介面上載入一件工程的基本資料。因此「選擇工程」這個功能在正式版中不成立。

**變更後定位**：本頁**不是專案儀表板，而是試算器**。所有計算結果必須**純粹由使用者手動輸入的「用料清單」與「時間設定」推導**，不依賴任何工程主檔。

> ⚠️ 此前提會連帶影響本頁多處元件，詳見 2.3 連帶影響總表。請優先確認此定位，再進行後續修改。

## 2.2　需求項目

### R-1　移除「選擇工程」區塊

- **現況**：左欄上方為工程選擇卡片，列出三個工程樣本。
- **問題**：功能不成立（見 2.1）。
- **需求**：
  - R-1.1　移除「選擇工程」整張卡片。
  - R-1.2　左欄版面重排，**「用料清單」升為本頁主角**，取得更大的視覺權重。

---

### R-2　用料清單改為「固定表單」

- **現況**：清單的材料項目由所選工程決定，因此表格會變形——高架橋顯示 5 項、鋼構橋 4 項、耐震補強僅 3 項。
- **問題**：
  1. 每次切換工程欄位數就改變，使用者無法建立固定的心智模型。
  2. 使用者**無法補上預設沒列、但實際有用到的材料**（例如耐震補強的清單中沒有 H 型鋼與砂石的輸入格），只能少估。
  3. 「未列出」與「用量填 0」語意混淆，看不出是「不使用該材料」還是「漏填」。
- **需求**：
  - R-2.1　**六項材料永遠全部列出、順序固定**：鋼筋、H型鋼、鋼板、預拌混凝土、瀝青混凝土、砂石。不因任何條件隱藏列。
  - R-2.2　用量**未填時顯示空白，不預設為 0**。需區分兩種狀態的語意：
    - 空白 ＝ 未填寫
    - 0 ＝ 確認不使用該材料
  - R-2.3　**單位欄唯讀、永遠顯示**（噸 / 方）。此為工程面最重要的防呆：瀝青混凝土為「元/噸」，預拌混凝土與砂石為「元/方」，不可要求使用者自行記憶。
  - R-2.4　**用量為空白或 0 的材料，不進入瀑布圖與成本走勢圖的圖例**。輸入表保留該列，圖表不呈現（否則六項中可能有三項貼在零軸，稀釋圖表資訊）。
  - R-2.5　用量數字顯示**千分位分隔**。

---

### R-3　新增「試算時間設定」：兩個時間端點並列且同級提升

- **現況**：
  - 原始編列時間：置於用料清單卡片底部，以一行小字加下拉選單呈現，且**選單約每三個月才給一個選項**，中間月份選不到。
  - 重估時間：不存在，鎖死在資料最新月份（民國 115/5）。
- **問題**：
  1. **時間軸不對稱**——一端可選、一端鎖死。既然全部計算都靠使用者輸入，此不對稱缺乏理由。使用者實務上會問「111/09 編列、112/06 開工，那時已經差多少」，現況無法回答。
  2. 兩個時間端點**共同**決定了本頁全部的計算結果，兩者是一組成對的輸入，不應一個藏在附註旁、一個不存在。
  3. 時間端點升為主要輸入後，選單跳月的問題就不可接受——使用者指定不到自己真正要的月份。
- **需求**：
  - R-3.1　新增**重估時間**選擇器，預設值為資料最新月份。
  - R-3.2　**原始編列時間與重估時間並列於同一個「試算時間設定」區塊**，兩者視覺層級相同，且與用量輸入同級。不得只提升其中一個。
  - R-3.3　三張 KPI 卡與瀑布圖的標題文案，須隨兩個時間點動態呈現，不再寫死「今日」。
  - R-3.4　**兩個時間選單皆須可選到「月」**——每一個有資料的月份都要是可選項，不得以季或其他間隔跳選。

### R-6　「建議材料預算」區塊改寫

- **現況**：右下角區塊標題為「規劃中類似工程 · 建議材料預算」，內容為「以今日物價重估 × (1 + 8%)」，理由為一句「過去一年多數材料仍在漲」。
- **問題**：
  1. 「類似工程」的前提隨工程主檔一同消失——沒有工程資料，就沒有相似性可言。
  2. 8% 緩衝為寫死的常數，但文案的寫法讓它看起來像是從資料推算出來的。
- **需求**：
  - R-6.1　標題與內文移除「類似工程」語意，改為以「本次試算之用料組合」為主體。
  - R-6.2　**緩衝比例的數值待需求方確認**（需求方傾向採用固定值）。開發時先以參數化方式保留此數值，待確認後填入。
  - R-6.3　若最終採固定值，**文案必須明確表達它是一個固定的編列緩衝，不得暗示該比例是由物價資料計算而來**。

### R-7　新增「用料結構占比」（圓餅圖）

- **背景**：改為純手動輸入後，**最大的風險從「資料錯誤」轉移到「打錯字」**。例如 18,000 方誤植為 1,800 方，算出的結果「看起來一樣合理」，介面完全沒有把關機制。現有瀑布圖只顯示各材料的**差額貢獻**，看不出用量的量級本身是否合理。
- **需求**：
  - R-7.1　於用料清單旁新增**用料結構占比**（各材料佔本次試算總成本的比例），**以圓餅圖呈現**。量級輸入錯誤時占比會明顯偏斜，可供使用者一眼察覺。
  - R-7.2　**須標示計算基準的時間點**。本頁有兩個時間端點（R-3），占比會隨採用的物價而不同。指定**以「重估時間」之物價計算**，並於圖表副標明確註記基準時間。
  - R-7.3　**顏色須沿用各材料既有的固定色**（與用料清單、瀑布圖、成本走勢圖一致），不得由圖表自行配色。
  - R-7.4　**扇形順序固定為用料清單的順序**（鋼筋、H型鋼、鋼板、預拌混凝土、瀝青混凝土、砂石），**不依占比大小重排**。目的是讓使用者能與左側輸入表逐列對照；若順序會跳動，對照功能即失效。
  - R-7.5　**每個扇形須顯示百分比數值**，不能只靠面積判讀。實際用料結構中常出現占比僅 2～4% 的材料（例如砂石），扇形極窄，無數值即無法閱讀。
  - R-7.6　用量為空白或 0 的材料不進入圓餅圖（同 R-2.4）。

### R-10　成本走勢圖須與時間設定連動

- **現況**：「同一份用料，成本如何隨物價變動」這張圖畫的是全部期間，**與使用者所選的兩個時間端點完全脫鉤**——線上沒有標出原始編列時點，也沒有標出重估時點。
- **問題**：R-3 把兩個時間端點提升為本頁的主要輸入之後，這張圖會是唯一一個對該輸入毫無反應的元件。使用者在 KPI 上看到「差額 +542 萬」，卻無法在走勢圖上指認那兩個點在哪裡，也就無法判斷這個差額是落在一段長期趨勢上，還是剛好卡在一個短期尖峰。
- **需求**：
  - R-10.1　圖上**明確標示「原始編列時間」與「重估時間」兩個時點**。
  - R-10.2　兩時點之間的區間須可視覺辨識，與區間外有所區別。
  - R-10.3　使用者變更任一時間端點時，圖上標示須同步更新。

## 4.1　需求項目

### D-1　刪除「成交」用詞

- **現況**：本頁有兩處使用「成交」，且全站僅此兩處：
  1. 單價走勢圖副標：「實際**成交**單價（元/噸）」
  2. 最新區域價卡片副標：「各區**成交**價」
- **問題**：本站資料來源為工程會價格趨勢，其性質並非實際成交結果。「成交」一詞誇大了資料的效力，可能讓使用者誤以為可據以核價或議價。
- **需求**：
  - D-1.1　移除上述兩處的「成交」。
  - D-1.2　**替代用詞定案為「單價」**，兩處一致：
    - 走勢圖副標：移除「成交」後僅保留單位標示（例：「單位：元/噸」）。
    - 最新區域價卡片副標：改為「**各區單價**」。

---

### D-2　刪除「12年區間」

- **現況**：走勢圖下方為四格統計列：最新單價 / 近一年 / 近三年 / 12年區間。
- **需求**：
  - D-2.1　移除「12年區間」該格。
  - D-2.2　**統計列由四格改為三格，須重新均分寬度**。現行版面為固定四欄，直接移除會留下空位。
- **備註（不影響本次開發，僅供日後參考）**：該格是本頁唯一提供「目前價格位於歷史區間何處」的元素，但它只呈現兩個裸數字（例如瀝青混凝土顯示 2,314–3,991），使用者無法從中看出目前的 3,932 其實已逼近歷史最高。換言之，被刪掉的是一個**表達失敗的元素**，不必然是一個沒有價值的概念。若日後仍想保留此資訊，較適當的位置是在走勢圖上以高低點參考線呈現，而非另闢一格數字。
