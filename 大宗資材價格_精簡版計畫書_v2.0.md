# 大宗資材價格資料自動下載與彙整計畫書（精簡版）

**版本** v2.0 ｜ 中華民國 115 年 7 月
**範疇** 自動化下載多來源大宗資材價格，彙整為單一乾淨資料表，供 Power BI 定時讀取
**與 v1.0 差異** 移除 n8n、Power BI REST API、service principal、Slack、品質報告關卡等重裝備；聚焦於「下載 + 彙整」單一程式

---

## 一、目標與範疇

### 目標
建立一支可自動化執行的程式，定時從**多個網路來源**下載各類大宗資材（鋼筋、H型鋼、鋼板、砂石、預拌混凝土、瀝青混凝土等）的價格資料，**自動彙整為單一、欄位一致的乾淨資料表**，輸出到固定位置，供 Power BI 自行定時讀取。

### 納入範圍
- 從多個網路來源自動下載各項資材的價格原始檔。
- 每月封存原始檔（因來源多為覆蓋式發布，不存即永久遺失歷史）。
- 將異質結構的多來源資料清洗、標準化、彙整為**單一輸出資料表**。
- 以 Windows 工作排程器定時執行整支程式。

### 排除範圍（不在本計畫）
- Power BI 的連線、建模、DAX、報表與刷新 — 由使用者自行處理。
- 程式僅負責產出乾淨資料表檔案；Power BI 如何讀取由使用者決定。

---

## 二、資料來源

各類資材同屬**政府資料開放平臺（data.gov.tw）**，但**每種資材是各自獨立的資料集、各有各的連結**。程式以「來源登錄檔（config）」管理這 7 條連結，**新增/調整資料集 = 改一筆設定，不需改動主程式**。來源清單如下（導引頁：工程會 `eid=7341`）：

| 資料集 | data.gov.tw 資料集頁 | 結構家族 |
|---|---|---|
| 大宗資材價格彙整表 | https://data.gov.tw/dataset/7374 | summary |
| 鋼筋 | https://data.gov.tw/dataset/6820 | wide_region |
| H型鋼 | https://data.gov.tw/dataset/6577 | wide_region |
| 鋼板 | https://data.gov.tw/dataset/6578 | wide_region |
| 砂石 | https://data.gov.tw/dataset/6818 | county_range |
| 混凝土（預拌） | https://data.gov.tw/dataset/6819 | region_stat |
| 瀝青混凝土 | https://data.gov.tw/dataset/6579 | region_stat |

> 官方網站（pcc.gov.tw / pcic.pcc.gov.tw）robots.txt 禁止自動爬取，故改採 data.gov.tw 之開放資料授權管道，適用「政府資料開放授權條款第 1 版」（免費、可加值、需標示來源）。各資料集頁底下提供 CSV 直連檔（主機路徑 `https://pcic.pcc.gov.tw/pwc-web/api/service/opendata-file/document/`），config 直接記錄各檔的下載網址。
> 更新頻率：每月一次，實際發布日不固定，約次月中旬。

## 三、核心設計原則（只保留必要的幾條）

1. **每月封存原始檔（不可變）**：來源覆蓋式發布、只留當期，程式每月將下載的原始檔存入以月份命名的資料夾（如 `data/raw/2026-07/`）。這是唯一無法省略的動作 — 不存，歷史永久消失。同月覆蓋即冪等，不需雜湊機制。
2. **來源設定驅動**：所有來源、結構家族、地區/單位對照集中在一個 config 檔，主程式對來源無知。
3. **重跑安全（冪等）**：同一份原始檔重跑，輸出資料表內容不變。
4. **單一乾淨輸出**：所有來源彙整為欄位一致的一張表（外加漲跌幅表與維度對照，見第五節），直接可被 Power BI 讀取，Power BI 端不需再轉換。

## 四、系統架構

單一語言、零伺服器、零雲端服務。

```
Windows 工作排程器（每月觸發）
        │
        ▼
  fetch.py  ──► 下載各來源原始檔 ──► data/raw/<YYYY-MM>/（不可變封存）
        │
        ▼
 consolidate.py ──► 依結構家族解析 → 清洗標準化 → 彙整 ──► data/curated/（輸出資料表）
        │
        ▼
   run.log（每次執行的簡易日誌：成功/略過/失敗）
        │
        ▼
   Power BI（使用者自行定時讀取 data/curated/）
```

| 層級 | 工具 | 說明 |
|---|---|---|
| 排程 | Windows 工作排程器 | 月更一次，內建即足夠，不需 n8n 或雲端 |
| 下載與封存 | Python（fetch.py） | 讀 config、逐來源下載、存入不可變 raw 層 |
| 清洗與彙整 | Python + pandas（consolidate.py） | 多結構解析、標準化、合併為單一表 |
| 輸出格式 | CSV（可選 Parquet） | Power BI 原生支援；CSV 便於人工檢視 |

> 若日後輸出需放到 OneDrive/SharePoint 供 Power BI 服務讀取，同一支程式改寫入該路徑即可，架構不變。

### 4.1 排程與「電腦未開機」的處理

資料為月更、發布日不固定，且程式冪等（重跑、跑多次結果不變），故**不需準時觸發，只需「當月有跑到」即可**。

- **方案 A（採用，全本機）**：Windows 工作排程器，每月觸發數次（例如每月 12、16、20、24 日），並勾選 **「在錯過排定的開始時間後，盡快啟動工作」**。
  - 排程當下電腦關機 → **下次開機自動補跑**。
  - 只要當月開過機一次即抓得到當月資料，重複執行因冪等而無害。
- **方案 B（備選，電腦可完全不開）**：改用 GitHub Actions 雲端排程，不依賴本機。代價是輸出檔須存到雲端（repo 或 OneDrive/SharePoint），Power BI 改從雲端讀取。適用於本機常態長期不開機的情境。

## 五、資料處理挑戰與對策

多來源異質資料，以下為彙整成單一表時必須處理的實際問題：

| 編號 | 問題 | 對策 |
|---|---|---|
| C-1 | 價格欄名內嵌民國年月（例：`114.8價格`），逐月變動 | 以正規表示式動態推斷欄位，不寫死欄名 |
| C-2 | 官方覆蓋式發布、無歷史 | 每月自行封存 raw 層，歷史由程式累積 |
| C-3 | 砂石價格為字串區間（例：`700~780`），全半形分隔符混用 | 拆為 price_min / price_max，取中點為代表值，標記 price_basis = range_mid |
| C-4 | 地區粒度不一（三區 / 加花蓮台東 / 不分區 / 22 縣市） | 保留原始地區標籤，另建 region_group、region_level 供彙總，避免撞鍵遺失 |
| C-5 | 彙整表價格與明細檔重複 | 彙整表僅取其獨有的跨期漲跌幅，輸出到獨立表，避免重複計價 |
| C-6 | 民國年月、千分位逗號、多種空值表示法（-、－、N/A、無、未調查…）、UTF-8/Big5 編碼交替、表尾註解列 | 於清洗程式統一正規化處理 |
| C-7 | **多來源結構不同，需併為一張表** | 每種結構家族一個解析器，統一轉成長格式（tidy）後縱向合併 |

## 六、輸出：交付給 Power BI 的資料表

程式輸出到 `data/curated/`，供 Power BI 讀取。以下為介面約定。

### 主表 fact_material_price（長格式，一列一個「期別×品項×地區」報價）

| 欄位群組 | 欄位 |
|---|---|
| 期別 | period, period_roc, period_date, snapshot |
| 品項 | material, item_name, unit, unit_assumed |
| 地區 | region, region_group, region_level |
| 價格 | price, price_min, price_max, price_basis |
| 統計 | std_dev, sample_size（僅預拌/瀝青混凝土提供） |
| 血緣 | source_dataset, source_file, ingest_ts |

### 漲跌幅表 fact_price_change（彙整表獨有的跨期漲跌）
period、item_name、region、horizon（半年/一年/兩年）、base_period、price_current、price_base、pct_change。

### 維度對照（選擇性輸出，或由 Power BI 端自建）
- dim_material：material、item_name、unit、unit_assumed
- dim_region：region、region_group、region_level
- dim_date：period_date、period、year、month、roc_year、quarter

> 輸出格式與位置為 **待確認事項 2**：預設 CSV、輸出到本機 `data/curated/`。若 Power BI 走服務端排程，改為 OneDrive/SharePoint 路徑或 Parquet。

## 七、錯誤處理與日誌（輕量）

不做品質關卡與外部告警，僅保留足以排查的最小機制：

| 情況 | 處理 |
|---|---|
| 某來源當月未發布（檔案不存在） | 記錄 missing，略過，繼續其他來源 |
| 某來源解析後為 0 列（疑似結構變更） | 記錄 warn，略過該來源，繼續 |
| 全部來源下載失敗 | 程式以非 0 退出碼結束，寫入 run.log |
| 正常 | 更新 curated 輸出，寫入 run.log |

排查方式：直接看 `run.log` 與 `data/raw/` 的封存檔。

## 八、歷史資料補齊（一次性、選擇性）

來源僅提供當期，過往無法回溯自動下載。若需要一開始就有趨勢：
- **一次性手動**從公共工程技術資料庫（`pcces.pcc.gov.tw`）之「重要資材趨勢價格」下載歷月彙整檔，放進對應的 `data/raw/<YYYY-MM>/`，再執行一次 consolidate.py（清洗邏輯完全相同，不需額外開發）。
- 之後由程式按月自動累積。

## 九、交付項目

| 項目 | 內容 |
|---|---|
| config/sources.yaml | 來源登錄檔：各來源網址、結構家族、地區與單位對照 |
| src/fetch.py | 下載與封存：讀 config、逐來源下載、存入不可變 raw 層 |
| src/consolidate.py | 清洗與彙整：多結構解析、標準化、合併輸出單一資料表 |
| data/raw/ | 每月原始檔封存（不可變） |
| data/curated/ | 輸出資料表（供 Power BI 讀取） |
| 工作排程器設定說明 | 每月自動執行的設定步驟 |
| README.md | 安裝、設定、操作說明 |

執行環境：Python 3.12、pandas、pyarrow、PyYAML、requests。
儲存空間：每月原始檔與輸出合計 < 1 MB，五年 < 100 MB。
維運：開發完成後每月約 0.5 小時（僅處理告警訊息）。

## 十、待確認事項

1. **輸出格式與位置**：CSV（本機）為預設；是否改 Parquet 或雲端路徑，取決於 Power BI 讀取方式（見第 4.1 節方案 A/B）。
2. **執行頻率**：來源發布日不固定（約次月中旬），預設每月排程數次並開啟「補跑錯過的工作」以確保抓到當月更新；請確認可接受。
3. **單位推定**：混凝土（元/立方公尺）、瀝青混凝土（元/公噸）之單位係依業界慣例填入，原始檔未提供，首次執行後應開檔核對。
4. **區域歸屬**：宜蘭縣預設歸北區，若貴單位慣例歸東區，改設定檔即可。
